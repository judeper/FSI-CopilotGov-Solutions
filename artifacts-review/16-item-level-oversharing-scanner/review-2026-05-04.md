#### Finding F1 — Restricted SharePoint Search caveat is missing

- **Severity:** Minor
- **Location:** line 93
- **Current text:** "Supports Restricted SharePoint Search planning by identifying overshared items that should be limited"
- **Issue:** Restricted SharePoint Search is a temporary, site-scoped discovery control and is not a security boundary. The current wording can be read as item-level limiting, but Microsoft documents RSS as an allowed-list control for SharePoint sites that does not change permissions.
- **Recommended correction:** "Supports temporary Restricted SharePoint Search or Restricted Content Discovery planning by identifying sites and items that need permission cleanup; RSS is site-scoped, short-term, and does not change permissions."
- **Source URL:** https://learn.microsoft.com/sharepoint/restricted-sharepoint-search
- **Source last updated:** 2026-02-24
- **Quote from source:** "Restricted SharePoint Search isn't a security boundary and doesn't change any permissions on SharePoint sites."

#### Finding F3 — Role guidance implies admin roles directly grant site content access

- **Severity:** Major
- **Location:** line 57
- **Current text:** "Confirm the operator has SharePoint Admin, Compliance Admin, or Global Admin rights as required for item-level permission enumeration."
- **Issue:** SharePoint Administrator and Global Administrator roles do not automatically grant access to all SharePoint sites or OneDrive content. Compliance roles are also not a substitute for target-site content access. Following this guidance can lead to failed item permission enumeration.
- **Recommended correction:** "Confirm target-site site collection administrator access for delegated enumeration, or document how a SharePoint Administrator/Global Administrator grants that access. Use Compliance Administrator/Purview roles only for Purview/DSPM review tasks."
- **Source URL:** https://learn.microsoft.com/sharepoint/sharepoint-admin-role
- **Source last updated:** 2025-05-29
- **Quote from source:** "Global Administrators and SharePoint Administrators don't have automatic access to all sites and each user's OneDrive"

#### Finding F4 — Graph app-only permission requirement is overprivileged/inaccurate

- **Severity:** Major
- **Location:** line 58
- **Current text:** "Confirm Sites.FullControl.All application permission is available if using app-only authentication."
- **Issue:** For the Microsoft Graph `driveItem` list permissions API, Microsoft documents least-privileged application permission as `Files.Read.All`, with `Files.ReadWrite.All`, `Sites.Read.All`, and `Sites.ReadWrite.All` as higher-privileged options. `Sites.FullControl.All` is not listed for this API. If PnP/SharePoint REST permissions are intended, they should be documented separately from Microsoft Graph permissions.
- **Recommended correction:** "For read-only Microsoft Graph permission enumeration, use documented least-privileged permissions such as `Files.Read.All` or `Sites.Read.All` where applicable; reserve write/full-control permissions for approved remediation and document the specific API surface."
- **Source URL:** https://learn.microsoft.com/graph/api/driveitem-list-permissions
- **Source last updated:** 2017-09-10 (ms.date; page updated_at 2025-07-23)
- **Quote from source:** "Application Files.Read.All Files.ReadWrite.All, Sites.Read.All, Sites.ReadWrite.All"

#### Finding F7 — Required administrative roles conflate compliance roles with SharePoint content access

- **Severity:** Major
- **Location:** lines 15-21
- **Current text:** "At least one of the following roles should be assigned ... SharePoint Admin; Compliance Admin; Global Admin ... For app-only authentication, the application registration requires `Sites.FullControl.All` permission in Microsoft Graph."
- **Issue:** The listed admin roles are not equivalent for item-level permission enumeration. SharePoint Administrator and Global Administrator do not automatically have access to all sites or OneDrive content, and Compliance Administrator is not a delegated SharePoint content-access role.
- **Recommended correction:** Separate prerequisites by task: target-site site collection administrator access for delegated enumeration; SharePoint Administrator or Global Administrator only to grant/manage access; Purview/Compliance Administrator roles for Purview/DSPM review; app-only permissions documented by API surface.

#### Finding F8 — Microsoft Graph permission list does not match the documented API

- **Severity:** Major
- **Location:** lines 38-43
- **Current text:** "When using app-only authentication with PnP PowerShell, the following Microsoft Graph API permissions are required: `Sites.FullControl.All`; `User.Read.All`."
- **Issue:** Microsoft Graph's `driveItem` list permissions API documents `Files.Read.All` as the least-privileged application permission and `Files.ReadWrite.All`, `Sites.Read.All`, and `Sites.ReadWrite.All` as higher-privileged permissions. It does not list `Sites.FullControl.All` or `User.Read.All` as required for listing item permissions. The current wording is overprivileged and mixes PnP/SharePoint and Graph permission models.
- **Recommended correction:** Document Graph read-only enumeration permissions according to the Graph API table, and separately document any SharePoint/PnP app-only permissions needed for live PnP operations or remediation.
- **Source URL:** https://learn.microsoft.com/graph/api/driveitem-list-permissions
- **Source last updated:** 2017-09-10 (ms.date; page updated_at 2025-07-23)
- **Quote from source:** "Choose the permission or permissions marked as least privileged for this API."

#### Finding F10 — Troubleshooting repeats overprivileged Graph permission guidance

- **Severity:** Major
- **Location:** lines 17, 46-47
- **Current text:** "For app-only authentication, verify that `Sites.FullControl.All` permission is granted ... Confirm `User.Read.All` Graph permission is available for user identity resolution"
- **Issue:** Microsoft Graph's item permission listing API does not require `Sites.FullControl.All` or `User.Read.All`; it documents `Files.Read.All` as least-privileged for application access, with `Files.ReadWrite.All`, `Sites.Read.All`, and `Sites.ReadWrite.All` as higher-privileged options. The troubleshooting guidance should distinguish Graph enumeration from PnP/SharePoint or remediation permissions.
- **Recommended correction:** "For Graph read-only permission listing, validate the documented Graph permissions for the selected endpoint; only require broader SharePoint/PnP or write permissions when the implementation performs approved remediation."
- **Source URL:** https://learn.microsoft.com/graph/api/driveitem-list-permissions
- **Source last updated:** 2017-09-10 (ms.date; page updated_at 2025-07-23)

### `solutions\08-license-governance-roi\README.md`

#### Finding F1 — Document budgets as alerts, not spending caps

- **Severity:** Major
- **Location:** lines 13 and 37
- **Current text:** "Beginning April 2026... cost cap monitoring through Azure Cost Management... Does not configure PAYG billing policies or spending caps..."
- **Issue:** Microsoft 365 Copilot pay-as-you-go budgets trigger notifications but do not enforce a cap or prevent spending beyond the budget. Calling them spending caps can mislead administrators.
- **Recommended correction:** "Replace cost cap/spending cap language with budget alert thresholds, and state that budgets do not enforce a hard cap."
- **Source URL:** https://learn.microsoft.com/microsoft-365/copilot/pay-as-you-go/setup
- **Source last updated:** 2025-11-18
- **Quote from source:** "the system doesn't enforce the budget or prevent your organization from exceeding it."

- **Location:** line 119
- **Current text:** "Microsoft Graph application permissions: `Reports.Read.All`, `Directory.Read.All`, `User.Read.All`."

- **Severity:** Critical

- **Issue:** The documented `getMicrosoft365CopilotUsageUserCounts` endpoint was not found, and current Microsoft 365 Copilot usage report APIs use `/v1.0/copilot/reports/...` or `/beta/copilot/reports/...` names such as UserCountSummary and UserCountTrend.

#### Finding F2 — Do not model per-user Copilot action counts as Graph report data

- **Severity:** Major
- **Location:** line 78
- **Current text:** "`CopilotUsageDetail` example fields include `CopilotActions30D`."
- **Issue:** Current Microsoft Learn notes that per-user Copilot prompt counts across tenants are not supported through the Graph report API; the API returns usage/last-activity detail, not a per-user action-count metric.
- **Recommended correction:** "Model last activity dates and active/enabled status from Graph reports; use admin-center exports, Purview audit, or clearly labeled local metrics for per-user prompt/action counts."
- **Source URL:** https://learn.microsoft.com/microsoft-365/copilot/extensibility/api/admin-settings/reports/copilotreportroot-getmicrosoft365copilotusageuserdetail
- **Source last updated:** 2026-02-19
- **Quote from source:** "Tracking per-user Copilot prompt counts across tenants isn't supported due to privacy and security constraints."

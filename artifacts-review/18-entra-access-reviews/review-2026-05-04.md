## Findings by file

### `solutions/18-entra-access-reviews/README.md`

#### Finding F1 — Access reviews are described as targeting SharePoint sites directly

- **Severity:** Major
- **Location:** lines 9, 67, 91, 98, 113
- **Current text:** "Automating Entra ID Access Reviews across SharePoint sites / SharePoint site members."
- **Issue:** Microsoft Graph access reviews attest access to Microsoft Entra resources such as groups, applications, access packages, and privileged roles. The README implies arbitrary SharePoint site membership can be reviewed directly.
- **Recommended correction:** "Scope the solution language to Microsoft 365 groups, security groups, or access packages that control SharePoint access, and state that direct SharePoint site permissions require a separate inventory/remediation pattern."
- **Source URL:** https://learn.microsoft.com/graph/api/resources/accessreviewsv2-overview
- **Source last updated:** 2024-06-18
- **Quote from source:** "The Microsoft Entra resources include groups, applications (service principals), access packages, and privileged roles."

#### Finding F2 — License prerequisite uses a non-current SKU name

- **Severity:** Major
- **Location:** line 47
- **Current text:** "Confirm Entra ID Governance P2 licensing is available for the target tenant."
- **Issue:** Microsoft Learn does not define an 'Entra ID Governance P2' SKU. Access reviews require Microsoft Entra ID Governance or Microsoft Entra Suite subscriptions, with some capabilities available under Microsoft Entra ID P2.
- **Recommended correction:** "Use 'Microsoft Entra ID Governance or Microsoft Entra Suite; Microsoft Entra ID P2 for applicable access review capabilities' and avoid the hybrid 'Governance P2' name."
- **Source URL:** https://learn.microsoft.com/entra/id-governance/access-reviews-overview
- **Source last updated:** 2026-03-12
- **Quote from source:** "This feature requires Microsoft Entra ID Governance or Microsoft Entra Suite subscriptions"

### `solutions/18-entra-access-reviews/docs/architecture.md`

#### Finding F1 — Architecture describes access reviews across SharePoint sites directly

- **Severity:** Major
- **Location:** lines 5, 46, 66
- **Current text:** "Automating Entra ID Access Reviews across SharePoint sites... Review scope targets SharePoint site members and guests."
- **Issue:** Microsoft Graph access review scopes are Microsoft Entra resources, not arbitrary SharePoint site membership. A SharePoint-focused workflow needs to bind reviews to groups or access packages that grant site access.

#### Finding F2 — Graph apply decisions endpoint is incorrect

- **Severity:** Critical
- **Location:** line 70
- **Current text:** "POST .../decisions/apply"
- **Issue:** The documented Microsoft Graph action is applyDecisions on the accessReviewInstance, not a decisions/apply child path. Following the architecture would produce a failed API call.
- **Recommended correction:** "Use POST /identityGovernance/accessReviews/definitions/{accessReviewScheduleDefinitionId}/instances/{accessReviewInstanceId}/applyDecisions."
- **Source URL:** https://learn.microsoft.com/graph/api/accessreviewinstance-applydecisions
- **Source last updated:** 2024-04-04
- **Quote from source:** "POST /identityGovernance/accessReviews/definitions/{accessReviewScheduleDefinitionId}/instances/{accessReviewInstanceId}/applyDecisions"

### `solutions/18-entra-access-reviews/scripts/Apply-ReviewDecisions.ps1`

#### Finding F1 — Script calls a non-existent apply decisions endpoint

- **Severity:** Critical
- **Location:** lines 7, 126-128
- **Current text:** "POST /identityGovernance/accessReviews/definitions/{id}/instances/{id}/decisions/apply"
- **Issue:** Microsoft Graph exposes applyDecisions as an action on accessReviewInstance. The current URI would fail for live execution.
- **Recommended correction:** "Change the URI to /identityGovernance/accessReviews/definitions/$DefinitionId/instances/$($instance.id)/applyDecisions."
- **Source URL:** https://learn.microsoft.com/graph/api/accessreviewinstance-applydecisions
- **Source last updated:** 2024-04-04
- **Quote from source:** "POST /identityGovernance/accessReviews/definitions/{accessReviewScheduleDefinitionId}/instances/{accessReviewInstanceId}/applyDecisions"

### `solutions/18-entra-access-reviews/scripts/New-AccessReview.ps1`

#### Finding F1 — absoluteMonthly recurrence omits required dayOfMonth

- **Severity:** Critical
- **Location:** lines 164-168
- **Current text:** "type = 'absoluteMonthly'; interval = [math]::Ceiling($cadence.frequencyDays / 30)"
- **Issue:** Microsoft Graph recurrencePattern requires dayOfMonth when type is absoluteMonthly. Live review creation can fail because the request body omits this required property.
- **Recommended correction:** "Add dayOfMonth, or use a supported weekly pattern where appropriate. For monthly/quarterly/semiannual reviews, set dayOfMonth to the intended review start day."
- **Source URL:** https://learn.microsoft.com/graph/api/resources/recurrencepattern
- **Source last updated:** 2024-08-08
- **Quote from source:** "The day of the month on which the event occurs. Required if type is absoluteMonthly"

#### Finding F2 — Graph scope does not match the documented SharePoint-and-guest review scope

- **Severity:** Major
- **Location:** lines 3, 8, 145-148, 190, 219
- **Current text:** "Creates Entra ID Access Reviews for SharePoint site members... scope = 'site-members-and-guests'; query = /groups/{groupId}/members"
- **Issue:** The live Graph body targets direct group members, not SharePoint site permissions, and it does not filter to guests. The documentation and output labels overstate the actual review scope.
- **Recommended correction:** "Rename the scope to group members, use /groups/{groupId}/transitiveMembers or the documented guest filter when intended, and explain how the group maps to SharePoint access."
- **Source URL:** https://learn.microsoft.com/graph/accessreviews-scope-concept
- **Source last updated:** 2025-08-19
- **Quote from source:** "query: /groups/{groupId}/transitiveMembers/?$filter=(userType eq 'Guest')"

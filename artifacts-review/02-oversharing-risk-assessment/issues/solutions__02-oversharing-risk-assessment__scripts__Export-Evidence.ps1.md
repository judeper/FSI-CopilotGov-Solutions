## Findings

### F1 — Sensitivity label recommendation overstates Copilot restriction

- **Severity:** Major
- **Location:** lines 244-245, 257
- **Current:** "Apply Microsoft Purview Information Protection sensitivity labels to all sites containing regulated data to restrict Microsoft 365 Copilot content surfacing."
- **Issue:** Applying labels is not by itself a blanket restriction on Microsoft 365 Copilot surfacing. Microsoft documents that AI apps honor protections such as encryption labels, where users need VIEW and EXTRACT rights, and labels must be configured for the intended protection.
- **Suggested change:** Change the recommendation to apply and configure sensitivity labels and protection settings for regulated data, and state that label coverage is a governance signal rather than an automatic Copilot surfacing restriction.
- **Source:** https://learn.microsoft.com/purview/ai-microsoft-purview (last updated 2026-05-01)
- **Quote:** "When the sensitivity label applies encryption, users must have the EXTRACT usage right, as well as VIEW."

### F2 — DSPM top-100 statement needs classic/default-assessment context

## Findings

### F1 — Power Automate connector blocking is governed by Power Platform data policies

- **Severity:** Major
- **Location:** line 53
- **Current:** "Confirm the environment type and Microsoft Purview Data Loss Prevention policies."
- **Issue:** For blocked Power Automate connectors in a Power Platform environment, the current admin surface is Power Platform data policies/DLP, not Microsoft Purview DLP policies. Microsoft Purview DLP protects sensitive data across Microsoft 365 and endpoints; Power Platform data policies govern connector use in apps, flows, and Copilot Studio.
- **Suggested change:** Change the action to confirm Power Platform data policies in the Power Platform admin center, and mention Microsoft Purview DLP only when discussing separate sensitive-data controls.
- **Source:** https://learn.microsoft.com/power-platform/admin/wp-data-loss-prevention (last updated 2026-04-07)
- **Quote:** "Power Platform admin center data policies allow administrators to control access to these connectors."

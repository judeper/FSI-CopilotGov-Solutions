## Findings

### F1 — License validation only models standalone SharePoint Advanced Management

- **Severity:** Major
- **Location:** lines 59-82
- **Current:** "Requirement = 'SharePoint Advanced Management' ... Validate SharePoint Advanced Management licensing in the tenant before production deployment."
- **Issue:** The deployment manifest records only a generic SharePoint Advanced Management requirement and does not model the current Microsoft 365 Copilot entitlement path for SharePoint Advanced Management features needed for Copilot deployment.
- **Suggested change:** Rename the check to validate SharePoint Advanced Management feature entitlement and record whether it is satisfied by Microsoft 365 Copilot licensing or standalone Microsoft SharePoint Advanced Management.
- **Source:** https://learn.microsoft.com/sharepoint/advanced-management (last updated 2026-04-23)
- **Quote:** "Microsoft 365 Copilot license: At least one user in your organization must be assigned a Copilot license."

### F2 — Restricted SharePoint Search status uses enforcement language

- **Severity:** Major
- **Location:** lines 112-138
- **Current:** "Status = if ($ScanMode -eq 'AutoRemediate') { 'planned-enforce' } else { 'planned-enable' }"
- **Issue:** Using planned-enforce for Restricted SharePoint Search can mislead operators because Microsoft states RSS is not a security boundary and does not change permissions.
- **Suggested change:** Use status names such as planned-temporary-enable and add notes that RSS is temporary, limited to 100 allowed sites, not a security boundary, and not a permission change.
- **Source:** https://learn.microsoft.com/sharepoint/restricted-sharepoint-search (last updated 2026-02-24)
- **Quote:** "Restricted SharePoint Search isn't a security boundary and doesn't change any permissions on SharePoint sites."

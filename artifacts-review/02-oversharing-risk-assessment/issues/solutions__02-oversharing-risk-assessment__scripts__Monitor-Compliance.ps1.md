## Findings

### F1 — Graph call uses a non-supported sensitivityLabel select instead of extractSensitivityLabels

- **Severity:** Critical
- **Location:** lines 689-700
- **Current:** "-Uri "/sites/$siteId/drive/root/children?`$select=id,sensitivityLabel&`$top=50" ... $_.sensitivityLabel"
- **Issue:** The script attempts to read a sensitivityLabel property from driveItem children. Current Microsoft Graph documentation provides a driveItem extractSensitivityLabels action for assigned sensitivity labels; relying on `$select=sensitivityLabel` for children will not produce the documented label extraction result.
- **Suggested change:** List drive items, then call `POST /sites/{site-id}/drive/items/{item-id}/extractSensitivityLabels` (or an equivalent documented driveItem path) and handle its `labels` response. Document the required Graph permissions such as Files.Read.All or Sites.Read.All.
- **Source:** https://learn.microsoft.com/graph/api/driveitem-extractsensitivitylabels?view=graph-rest-1.0 (last updated 2024-04-04)
- **Quote:** "POST /sites/{site-id}/drive/items/{item-id}/extractSensitivityLabels"

### F2 — Sensitivity label recommendation overstates Copilot restriction

- **Severity:** Major
- **Location:** lines 664, 731
- **Current:** "Apply Microsoft Purview Information Protection sensitivity labels to all sites containing regulated data to restrict Microsoft 365 Copilot content surfacing."
- **Issue:** The recommendation implies sensitivity labels automatically restrict Copilot content surfacing. Microsoft states Copilot honors label-based protections such as encryption, where the user must have VIEW and EXTRACT rights; labels must be configured for the intended protection.
- **Suggested change:** Change the recommendation to configure sensitivity labels and protection settings for regulated data and avoid implying label coverage alone blocks Copilot surfacing.
- **Source:** https://learn.microsoft.com/purview/ai-microsoft-purview (last updated 2026-05-01)
- **Quote:** "When the sensitivity label applies encryption, users must have the EXTRACT usage right, as well as VIEW."

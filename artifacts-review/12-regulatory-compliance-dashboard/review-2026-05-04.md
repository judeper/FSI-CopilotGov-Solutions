## Findings by file

### `solutions/12-regulatory-compliance-dashboard/README.md`

#### Finding F1 — Use current Power BI semantic model terminology

- **Severity:** Major
- **Location:** lines 9, 46, 54, 64, 68, 78
- **Current text:** "Power BI dataset specifications"; "Power BI dataset tables"; "configure dataset refresh"
- **Issue:** Microsoft Learn now uses **semantic model** for the Power BI data model artifact. Continuing to use user-facing "dataset" terminology can confuse implementers following current Power BI/Fabric guidance.
- **Recommended correction:** Replace user-facing "Power BI dataset" references with "Power BI semantic model" and use "semantic model refresh" where the text describes refresh configuration.
- **Source URL:** https://learn.microsoft.com/power-bi/connect-data/service-datasets-understand
- **Source last updated:** 2025-10-01
- **Quote from source:** "Power BI semantic models represent a source of data that's ready for reporting and visualization."

#### Finding F2 — Replace outdated Power BI Admin role reference

- **Severity:** Major
- **Location:** line 86
- **Current text:** "Power BI Admin and Dataverse System Administrator permissions for deployment operators."
- **Issue:** The current Microsoft Learn admin role guidance names **Fabric administrator** or **Power Platform administrator** for organization-wide Fabric/Power BI administration. "Power BI Admin" is outdated/imprecise as a role name.
- **Recommended correction:** "Fabric administrator or Power Platform administrator for tenant-wide Fabric/Power BI settings, Power BI workspace Admin rights for workspace operations, and Dataverse System Administrator/System Customizer permissions as required."
- **Source URL:** https://learn.microsoft.com/power-bi/admin/service-admin-role
- **Source last updated:** 2026-04-21
- **Quote from source:** "Power Platform administrator or Fabric administrator"

#### Finding F3 — Use current Microsoft Purview Suite licensing name

- **Severity:** Major
- **Location:** line 87
- **Current text:** "Microsoft 365 E5 Compliance if Purview-backed evidence references are required."
- **Issue:** Microsoft Learn identifies the current SKU name as **Microsoft Purview Suite** and describes Microsoft 365 E5 Compliance as the former name. The prerequisite should use the current licensing name and avoid implying that Compliance Manager access always requires that add-on.
- **Recommended correction:** "Microsoft Purview Suite, Microsoft 365 E5/A5/G5, or the eligible Purview add-on/licensing required for the specific Purview Compliance Manager templates or evidence features in use."
- **Source URL:** https://learn.microsoft.com/purview/purview-trial
- **Source last updated:** 2026-03-30
- **Quote from source:** "Microsoft Purview Suite (formerly known as Microsoft 365 E5 Compliance)"

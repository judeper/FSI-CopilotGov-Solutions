### `solutions/21-cross-tenant-agent-federation-auditor/README.md`

#### Finding F1 — Copilot Studio publishing modes are not current product terms

- **Severity:** Major
- **Location:** lines 15, 24
- **Current text:** "Cross-tenant Copilot agent invocation patterns published from Copilot Studio (multi-tenant and tenant-restricted authoring modes)."
- **Issue:** Microsoft Learn describes Copilot Studio publishing through channels and authentication/sharing settings. I could not verify `tenant-restricted authoring modes` or `restricted-multi-tenant` as current Copilot Studio publishing modes; the provided `multi-tenant` Learn hint returned 404.
- **Recommended correction:** Replace this with language such as "Copilot Studio agent channels, authentication settings, and organization sharing controls that influence who can chat with an agent."
- **Source URL:** https://learn.microsoft.com/en-us/microsoft-copilot-studio/publication-fundamentals-publish-channels
- **Source last updated:** 2026-04-24
- **Quote from source:** "You can publish your agent on multiple platforms, or channels."

#### Finding F2 — Agent ID federated-trust and signing-key terminology is over-specific

- **Severity:** Major
- **Location:** lines 16, 27, 74
- **Current text:** "Microsoft Entra Agent ID federated trust relationships, including issuer, audience, and signing key metadata recorded for each agent identity."
- **Issue:** Current Agent ID documentation describes agent identities, blueprints, ownership/sponsorship, Conditional Access, and audit logs. It does not document an Agent ID feature named "federated trust relationships" with issuer/audience/signing-key metadata or an Agent ID signing/key-rotation control surface as written.
- **Recommended correction:** Recast these as customer-defined attestation fields, or align them to documented Agent ID resources such as agent identity, agent identity blueprint, owner, sponsor, Conditional Access posture, and sign-in/audit logs.
- **Source URL:** https://learn.microsoft.com/en-us/graph/api/resources/agentid-platform-overview
- **Source last updated:** 2026-03-14
- **Quote from source:** "The following core components comprise Microsoft Entra Agent ID's architecture:"

#### Finding F3 — MCP attestation and signing-key thumbprint claims are not Learn-supported

- **Severity:** Major
- **Location:** lines 17, 26, 38, 73
- **Current text:** "MCP server federated trust attestations (server identity, transport, scopes, and attestation freshness)" and "signing-key thumbprint".
- **Issue:** Microsoft Learn documents connecting agents to MCP servers, configuring authentication, approving tool calls, and using allow lists. It does not document a Microsoft MCP server attestation mechanism with signing-key thumbprints or attestation freshness.

#### Finding F2 — Cross-tenant access policy read scopes are vague and incomplete

- **Severity:** Major
- **Location:** line 14
- **Current text:** "Entra app registration with delegated read scopes for cross-tenant access policies (least privilege)."
- **Issue:** The prerequisite does not name the actual Microsoft Graph permission. The v1.0 List partners API supports `Policy.Read.All` for delegated and application access, with `Policy.ReadWrite.CrossTenantAccess` as higher privilege.
- **Recommended correction:** Specify `Policy.Read.All` as least privilege for read-only enumeration and document supported delegated roles such as Security Reader, Global Reader, Global Secure Access Administrator, Teams Administrator, or Security Administrator as applicable.

- **Source last updated:** 2025-07-23
- **Quote from source:** "Policy.Read.All"

#### Finding F3 — Role names are not canonical and omit current Agent ID/AI roles

- **Severity:** Major
- **Location:** lines 20-24
- **Current text:** "Entra Global Admin", "Purview Compliance Admin", and "M365 Global Admin".
- **Issue:** The page says to use canonical short names, but these are not the canonical Microsoft Entra role names in Learn. Current roles include Global Administrator, Compliance Administrator, Agent ID Administrator, Agent ID Developer, AI Administrator, and AI Reader.

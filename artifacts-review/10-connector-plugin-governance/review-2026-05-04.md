### `solutions/10-connector-plugin-governance/README.md`

#### Finding F1 — Global Administrator is overstated for agent/plugin governance

- **Severity:** Major
- **Location:** line 116
- **Current text:** "- Microsoft 365 Global Admin access for Teams app policy and plugin deployment review."
- **Issue:** Current Microsoft 365 admin center agent-management documentation allows AI Administrator as a tenant-wide governance role. Stating Global Admin as required overstates the privilege needed and conflicts with least-privilege guidance.
- **Recommended correction:** "Use AI Administrator as the preferred role for Microsoft 365 admin center agent/plugin governance, with Global Administrator reserved only where an operation explicitly requires it."
- **Source URL:** https://learn.microsoft.com/microsoft-365/admin/manage/agent-roles-perms?view=o365-worldwide
- **Source last updated:** 2026-04-21
- **Quote from source:** "The AI Administrator and Global Administrator roles have tenant-wide visibility and governance authority."

#### Finding F2 — Copilot Studio extensibility should use current tools/actions terminology

- **Severity:** Minor
- **Location:** line 150
- **Current text:** "- Copilot Studio plugins and declarative agents can require separate tenant and Teams app policy configuration outside this solution package."
- **Issue:** Current Copilot Studio documentation describes these extensions as tools, including connectors, REST APIs, MCP, prompts, and agent flows. "Copilot Studio plugins" is stale/imprecise terminology for the current authoring surface.
- **Recommended correction:** "Replace "Copilot Studio plugins" with "Copilot Studio tools/actions" and keep "plugins" for Microsoft 365 Copilot declarative-agent plugin packages where applicable."
- **Source URL:** https://learn.microsoft.com/microsoft-copilot-studio/advanced-plugin-actions
- **Source last updated:** 2026-01-29
- **Quote from source:** "You can extend the capabilities of your custom agent by adding one or more tools."

#### Finding F3 — Microsoft Graph plugin inventory needs Agent Registry preview caveat

- **Severity:** Major
- **Location:** line 151
- **Current text:** "- Microsoft AppSource metadata and Microsoft Graph inventory can require manual reconciliation for custom plugin publishers."
- **Issue:** The text implies generic Microsoft Graph inventory covers custom plugin publishers. Current documentation identifies Agent Registry and Agent Details Microsoft Graph APIs as preview for programmatic agent inventory; app-registration inventory alone will not provide full agent/tool metadata.
- **Recommended correction:** "Refer to Microsoft 365 admin center Agent Registry/Data & tools metadata, and if programmatic access is used, call out the Microsoft Graph Agent Registry/Package APIs as preview and role-bound to AI Admin."
- **Source URL:** https://learn.microsoft.com/microsoft-365/admin/manage/agent-registry?view=o365-worldwide
- **Source last updated:** 2026-05-01
- **Quote from source:** "Microsoft Graph API for Agent Registry and Agent Details (preview)."

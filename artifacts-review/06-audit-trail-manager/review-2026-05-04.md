### `solutions/06-audit-trail-manager/DELIVERY-CHECKLIST.md`

#### Finding F1 — Bare AIInteraction is not a current user-interaction audit operation

- **Severity:** Major
- **Location:** line 8
- **Current text:** "`CopilotInteraction` and `AIInteraction` events are confirmed in the validation window."
- **Issue:** Microsoft Learn identifies Copilot and AI user interaction operation/record type values as CopilotInteraction, ConnectedAIAppInteraction, and AIAppInteraction. A bare AIInteraction event name is not the documented operation for Copilot user interactions.
- **Recommended correction:** "Use CopilotInteraction for Microsoft-developed Copilot interactions. If the scope intentionally includes custom or third-party AI apps, add ConnectedAIAppInteraction or AIAppInteraction; remove bare AIInteraction."
- **Source URL:** https://learn.microsoft.com/en-us/purview/audit-copilot
- **Source last updated:** 2026-04-15
- **Quote from source:** "For user interactions with Copilot, this property uses values like *CopilotInteraction*, *ConnectedAIAppInteraction*, and *AIAppInteraction*"

#### Finding F2 — SharePointFileAccess is not the documented SharePoint file audit operation

- **Severity:** Major
- **Location:** line 9
- **Current text:** "Required supporting events such as `SharePointFileAccess` are included in the monitoring scope."
- **Issue:** Microsoft Learn documents SharePoint/OneDrive file access under FileAccessed and related operations, not SharePointFileAccess. Using the undocumented operation name can cause audit searches or retention-policy filters to miss records.
- **Recommended correction:** "Replace SharePointFileAccess with FileAccessed, and include FileAccessedExtended only when extended repeated-access events are in scope."
- **Source URL:** https://learn.microsoft.com/en-us/purview/audit-log-activities
- **Source last updated:** 2026-04-02
- **Quote from source:** "Accessed file | FileAccessed | User or system account accesses a file."

#### Finding F3 — Use Audit (Premium) terminology instead of Advanced audit level

- **Severity:** Major
- **Location:** line 10
- **Current text:** "Audit level for the selected tier is documented as Standard or Advanced as required."
- **Issue:** Current Microsoft Purview audit tiers are Audit (Standard) and Audit (Premium). Describing the tier as Standard or Advanced uses legacy/imprecise terminology and can confuse licensing and configuration expectations.
- **Recommended correction:** "Rename Advanced audit level references to Audit (Premium), and model tier values as Audit (Standard) / Audit (Premium)."
- **Source URL:** https://learn.microsoft.com/en-us/purview/audit-solutions-overview
- **Source last updated:** 2026-02-18
- **Quote from source:** "Audit (Premium) includes all Audit (Standard) functionality."

#### Finding F4 — Use AuditLogsQuery permissions for the Audit Search Graph API

- **Severity:** Major
- **Location:** line 56
- **Current text:** "Graph application permissions or delegated permissions approved for AuditLog.Read.All and RecordsManagement.Read.All."
- **Issue:** The Microsoft Graph Audit Search API uses AuditLogsQuery permissions, including AuditLogsQuery.Read.All or service-specific AuditLogsQuery-* scopes. AuditLog.Read.All and RecordsManagement.Read.All do not match the documented Audit Search API permission model.
- **Recommended correction:** "Replace the audit-search prerequisite with AuditLogsQuery.Read.All or the least-privileged service-specific AuditLogsQuery-* permission required for the selected workload; document RecordsManagement permissions separately only where records APIs are used."
- **Source URL:** https://learn.microsoft.com/en-us/graph/api/security-auditcoreroot-list-auditlogqueries?view=graph-rest-1.0&preserve-view=true
- **Source last updated:** 2024-10-29
- **Quote from source:** "AuditLogsQuery.Read.All"

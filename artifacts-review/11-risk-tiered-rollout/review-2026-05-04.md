## Findings by file
### `solutions/11-risk-tiered-rollout/README.md`

#### Finding F1 — Graph license-assignment permissions are incomplete for group-based rollout

- **Severity:** Major
- **Location:** lines 101-103
- **Current text:** "Required Microsoft Graph permissions are approved: `User.ReadWrite.All` and `Directory.Read.All`."
- **Issue:** The prerequisite summary lists `User.ReadWrite.All` and read-only `Directory.Read.All`, but the Microsoft Graph group assignLicense API uses `LicenseAssignment.ReadWrite.All` as the least-privileged delegated/application permission. Delegated group assignment also requires a supported Microsoft Entra role.
- **Recommended correction:** "Replace the summary with `LicenseAssignment.ReadWrite.All` for license assignment and document the supported delegated role requirement. If implementation uses group-based assignment, reference `POST /groups/{id}/assignLicense`; if it uses direct assignment, reference `POST /users/{id | userPrincipalName}/assignLicense`."
- **Source URL:** https://learn.microsoft.com/graph/api/group-assignlicense
- **Source last updated:** 2024-10-15
- **Quote from source:** "Delegated (work or school account) LicenseAssignment.ReadWrite.All"
#### Finding F2 — Usage location prerequisite is missing for license-assignment waves

- **Severity:** Minor
- **Location:** lines 99-104
- **Current text:** "Microsoft 365 E3 or E5 licensing plus Microsoft 365 Copilot licenses are available for the intended wave size."
- **Issue:** For Microsoft Entra group-based license assignment, Microsoft documents that usage location should be set before assigning a license. Omitting this caveat can confuse multi-location tenants and lead to incorrect inherited locations.
- **Recommended correction:** "Add a prerequisite that the Usage location property is populated for all users before adding them to wave-based license-assignment groups."
- **Source URL:** https://learn.microsoft.com/entra/identity/users/licensing-group-advanced
- **Source last updated:** 2025-01-15
- **Quote from source:** "Before a license can be assigned to a user, the administrator should specify the Usage location property on the user."
### `solutions/11-risk-tiered-rollout/config/default-config.json`

#### Finding F3 — Microsoft 365 Copilot SKU part number does not match Microsoft licensing reference

- **Severity:** Major
- **Location:** line 173
- **Current text:** ""skuPartNumber": "MICROSOFT_365_COPILOT""
- **Issue:** The configured `skuPartNumber` uses an all-uppercase value. Microsoft Learn lists the Copilot SKU part number as `Microsoft_365_Copilot`; exact matching against `subscribedSku.skuPartNumber` can fail or generate incorrect assignment manifests.
- **Recommended correction:** "Change the configured SKU part number to `Microsoft_365_Copilot`, or document lookup by `skuId` from `GET /subscribedSkus` before building assignment actions."
- **Source URL:** https://learn.microsoft.com/entra/identity/users/licensing-service-plan-reference
- **Source last updated:** 2026-03-16
- **Quote from source:** "Copilot for Microsoft 365 Microsoft_365_Copilot 639dec6b-bb19-468b-871c-c5c441c4b0cb"
### `solutions/11-risk-tiered-rollout/docs/prerequisites.md`

#### Finding F4 — Graph permission guidance omits least-privileged scope and delegated role requirements

- **Severity:** Major
- **Location:** lines 17-20
- **Current text:** "`User.ReadWrite.All` (covers license assignment via `POST /users/{id}/assignLicense`) and `Directory.Read.All`."
- **Issue:** `User.ReadWrite.All` is documented as a higher-privileged permission for direct user license assignment, while `LicenseAssignment.ReadWrite.All` is the least-privileged permission. The file also omits the supported Microsoft Entra roles required for delegated calls.
- **Recommended correction:** "Use `LicenseAssignment.ReadWrite.All` as the baseline permission, keep higher-privileged alternatives only as optional, and document delegated roles such as Directory Writers, License Administrator, or User Administrator for direct user assignment. Add group-assignment roles if using `/groups/{id}/assignLicense`."
- **Source URL:** https://learn.microsoft.com/graph/api/user-assignlicense
- **Source last updated:** 2024-10-15
- **Quote from source:** "Delegated (work or school account) LicenseAssignment.ReadWrite.All"
#### Finding F5 — Usage location prerequisite is missing for group-based licensing

- **Severity:** Minor
- **Location:** lines 9-20
- **Current text:** "Licensing and Graph permissions prerequisites do not mention the user Usage location property."
- **Issue:** Microsoft Entra group-based licensing guidance states that usage location should be specified before license assignment. This is an important prerequisite for tenants with users in multiple locations.
- **Recommended correction:** "Add a licensing prerequisite requiring the Usage location property to be populated for users before wave-based group license assignment."
- **Source URL:** https://learn.microsoft.com/entra/identity/users/licensing-group-advanced
- **Source last updated:** 2025-01-15
- **Quote from source:** "Before a license can be assigned to a user, the administrator should specify the Usage location property on the user."

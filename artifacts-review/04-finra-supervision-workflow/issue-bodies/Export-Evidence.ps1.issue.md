## Findings

### F1 — Dataverse Web API queries use table logical names instead of EntitySet names

- **Severity:** Critical
- **Location:** lines 279 and 356-358
- **Current:** "`$uri = '{0}/api/data/v9.2/{1}?$select={2}'`" with table names `fsi_cg_fsw_queue`, `fsi_cg_fsw_log`, and `fsi_cg_fsw_config`.
- **Issue:** Dataverse Web API collection paths require the environment's EntitySet resource name, which is usually plural and can differ from the table logical name. Using logical names directly can cause live export requests to fail.
- **Suggested change:** Resolve EntitySet names from the Dataverse service document or make them explicit configuration values, then call `/api/data/v9.2/{entitySetName}`.
- **Source:** https://learn.microsoft.com/power-apps/developer/data-platform/webapi/query/overview (last updated 2026-01-09)
- **Quote:** "These values are usually the plural name of the table, but they can be different."

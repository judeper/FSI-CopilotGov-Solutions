## Findings

### F1 — Licensing section uses former Microsoft 365 E5 Compliance name as the required license

- **Severity:** Major
- **Location:** lines 9-10
- **Current:** "Microsoft 365 E5 Compliance - Required for Microsoft Purview Communication Compliance policies that target Copilot prompts and responses."
- **Issue:** Microsoft Learn identifies Microsoft Purview Suite as the current name and lists other eligible plans/add-ons; stating only Microsoft 365 E5 Compliance as required is outdated and overly narrow.
- **Suggested change:** "Microsoft Purview Suite (formerly Microsoft 365 E5 Compliance), Office 365 Enterprise E5, or Office 365 Enterprise E3 with the Advanced Compliance add-on for users covered by Communication Compliance policies."
- **Source:** https://learn.microsoft.com/purview/communication-compliance-plan (last updated 2025-08-12)
- **Quote:** "Office 365 Enterprise E3 license with the Advanced Compliance add-on"

### F2 — Role prerequisites use a non-current Communication Compliance admin role label

- **Severity:** Major
- **Location:** lines 14-17
- **Current:** "Purview Compliance Admin - Configures and validates Microsoft Purview Communication Compliance policies and reviewer scope."
- **Issue:** The role label should match Microsoft Learn's Communication Compliance role groups or Compliance Administrator roles/role groups.
- **Suggested change:** Use "Communication Compliance Admins role group" for policy/settings administration, or "Microsoft Entra Compliance Administrator" / "Microsoft Purview portal Compliance Administrator" where broad compliance administration is approved.
- **Source:** https://learn.microsoft.com/purview/communication-compliance-permissions (last updated 2025-08-14)
- **Quote:** "_Communication Compliance Admins_ role group"

### F3 — Graph API permissions list wrong audit-search permission name

- **Severity:** Major
- **Location:** lines 34-41
- **Current:** "Typical permissions... `Policy.Read.All`, `AuditLog.Read.All`, `User.Read.All`."
- **Issue:** Microsoft Graph Audit Search API uses `AuditLogsQuery-*` permissions, not `AuditLog.Read.All`. `Policy.Read.All` is not documented as an audit-search permission for `/security/auditLog/queries`.
- **Suggested change:** For Audit Search Graph API scenarios, use `AuditLogsQuery.Read.All` or least-privileged service-specific permissions such as `AuditLogsQuery-Entra.Read.All`, plus separate Graph permissions only where needed for user/group lookup.
- **Source:** https://learn.microsoft.com/graph/api/security-auditcoreroot-list-auditlogqueries?view=graph-rest-1.0 (last updated 2024-10-29)
- **Quote:** "AuditLogsQuery.Read.All"

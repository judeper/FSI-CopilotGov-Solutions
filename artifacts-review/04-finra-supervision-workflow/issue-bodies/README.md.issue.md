## Findings

### F1 — Communication Compliance licensing uses former SKU name

- **Severity:** Major
- **Location:** line 53
- **Current:** "Microsoft 365 E5 Compliance for Microsoft Purview Communication Compliance signals."
- **Issue:** Microsoft Learn now describes the license as Microsoft Purview Suite, formerly Microsoft 365 E5 Compliance, and also documents eligible Office 365 E5 / E3 plus Advanced Compliance options.
- **Suggested change:** "Microsoft Purview Suite (formerly Microsoft 365 E5 Compliance), Office 365 Enterprise E5, or Office 365 Enterprise E3 with the Advanced Compliance add-on for Microsoft Purview Communication Compliance coverage."
- **Source:** https://learn.microsoft.com/purview/communication-compliance-plan (last updated 2025-08-12)
- **Quote:** "Microsoft Purview Suite (formerly known as Microsoft 365 E5 Compliance) license"

### F2 — Communication Compliance admin role name is not current

- **Severity:** Major
- **Location:** line 54
- **Current:** "Power Platform Admin, Purview Compliance Admin, and Global Reader access for deployment validation."
- **Issue:** "Purview Compliance Admin" is not the role/role-group name used by Microsoft Learn for Communication Compliance.
- **Suggested change:** Replace "Purview Compliance Admin" with "Communication Compliance Admins role group or Microsoft Entra Compliance Administrator / Microsoft Purview portal Compliance Administrator role group, as approved for least privilege."
- **Source:** https://learn.microsoft.com/purview/communication-compliance-permissions (last updated 2025-08-14)
- **Quote:** "_Communication Compliance Admins_ role group"

### F3 — Ingestion flow describes an unsupported scheduled Purview polling pattern

- **Severity:** Major
- **Location:** lines 78-80 and 100
- **Current:** "Trigger: scheduled poll or event-driven connector action that reads flagged Copilot prompt and response items from Microsoft Purview Communication Compliance."
- **Issue:** Microsoft Learn documents Power Automate in Communication Compliance as alert/user automation launched from an alert, not as a scheduled/event connector that polls flagged policy-match items.
- **Suggested change:** Document that ingestion requires a customer-validated supported export/integration pattern, such as Communication Compliance report exports, audit-log review, or a Power Automate flow launched from a Communication Compliance alert.
- **Source:** https://learn.microsoft.com/purview/communication-compliance-power-automate (last updated 2025-04-07)
- **Quote:** "working directly in an alert"

## Findings by file

### `solutions/19-copilot-tuning-governance/README.md`

#### Finding F1 — Copilot Tuning preview status is omitted

- **Severity:** Major
- **Location:** lines 9, 13, 57, 94-97
- **Current text:** "This solution provides a framework for governing Microsoft 365 Copilot Tuning, a feature available to organizations with 5,000 or more Copilot licenses..." and "Deploy this solution before enabling Copilot Tuning in any tenant."
- **Issue:** Microsoft Learn describes Copilot Tuning as an early access preview available to a limited set of customers, not a generally available feature that can be planned solely from license count. The README should disclose preview status and requirements subject to change wherever it describes availability and tier progression.
- **Recommended correction:** State that Microsoft 365 Copilot Tuning is currently an early access preview/limited customer capability; during public preview, tenants with at least 5,000 Microsoft 365 Copilot licenses are eligible, but access and requirements are subject to change.
- **Source URL:** https://learn.microsoft.com/en-us/microsoft-365/copilot/copilot-tuning-overview
- **Source last updated:** 2026-04-10
- **Quote from source:** "currently available to a limited set of customers through early access programs"

#### Finding F2 — README implies public Copilot Tuning APIs exist

- **Severity:** Major
- **Location:** lines 29, 96
- **Current text:** "Does not connect to M365 Admin Center or Copilot Tuning APIs" and "until tenant-specific Copilot Tuning API calls are connected."
- **Issue:** Microsoft Learn documents Copilot Tuning administration through the Copilot control system in the Microsoft 365 admin center; it does not document a public "Copilot Tuning API" surface for these tasks. The wording could send implementers toward a non-existent endpoint.
- **Recommended correction:** Replace references to "Copilot Tuning APIs" with "Microsoft 365 admin center Copilot control system, Agent Builder, or Agent 365 integration surfaces" unless Microsoft publishes a Copilot Tuning API.
- **Source URL:** https://learn.microsoft.com/en-us/microsoft-365/copilot/copilot-tuning-admin-guide
- **Source last updated:** 2026-04-13
- **Quote from source:** "AI admins manage Copilot Tuning through the Copilot control system in the Microsoft 365 admin center."

### `solutions/19-copilot-tuning-governance/docs/architecture.md`

#### Finding F1 — Architecture treats the 5,000-license threshold as sufficient eligibility

- **Severity:** Major
- **Location:** lines 5, 101-105
- **Current text:** "allows organizations with 5,000 or more Copilot licenses to create fine-tuned AI agents from proprietary data" and tier rows with `Tuning Enabled` set to `Yes`.
- **Issue:** Current Microsoft Learn says Copilot Tuning is early access preview and limited to customers in early access programs; the 5,000 Microsoft 365 Copilot license threshold applies during public preview and does not by itself guarantee availability.
- **Recommended correction:** Add preview/limited-access language to the purpose and tier table, and describe tuning-enabled tiers as governance targets for eligible preview tenants rather than generally available tenant states.
- **Source URL:** https://learn.microsoft.com/en-us/microsoft-365/copilot/copilot-tuning-admin-guide
- **Source last updated:** 2026-04-13
- **Quote from source:** "During public preview, only tenants with at least 5,000 Microsoft 365 Copilot licenses are eligible."

#### Finding F2 — Training data flow omits SharePoint snapshot and policy caveats

- **Severity:** Major
- **Location:** lines 12-14, 66-72, 78-89
- **Current text:** "identifies source data" and "Source Data: Identification of the proprietary data to be used for tuning."
- **Issue:** The architecture describes source data generically, but Learn states that Copilot Tuning uses explicitly selected SharePoint content as training data and creates snapshot data. It also states that source DLP and retention policies do not automatically apply to snapshot data. These caveats are central to a governance architecture.
- **Recommended correction:** Add a training-data/snapshot component that records selected SharePoint sources, ACL capture time, sensitivity-label review, snapshot retention, and the fact that original SharePoint DLP/retention policies do not automatically apply to snapshot data.
- **Source URL:** https://learn.microsoft.com/en-us/microsoft-365/copilot/copilot-tuning-admin-guide
- **Source last updated:** 2026-04-13
- **Quote from source:** "only the SharePoint content explicitly selected by the user is used as training data"

### `solutions/19-copilot-tuning-governance/DELIVERY-CHECKLIST.md`

#### Finding F1 — Delivery checklist omits early access preview eligibility

- **Severity:** Major
- **Location:** lines 49-57
- **Current text:** "Verify the organization has 5,000 or more Copilot licenses, meeting the Copilot Tuning eligibility threshold."
- **Issue:** The checklist treats 5,000 licenses as the operative eligibility test. Microsoft Learn says Copilot Tuning is currently limited to early access programs and that features and requirements are subject to change.

- **Source URL:** https://learn.microsoft.com/en-us/microsoft-365/copilot/copilot-tuning-overview
- **Source last updated:** 2026-04-10
- **Quote from source:** "Features and requirements are subject to change."

#### Finding F2 — Deployment sequence assumes tuning starts disabled and ignores default-enabled eligible tenants

- **Severity:** Major
- **Location:** lines 48-64, 111-117
- **Current text:** "Start with the baseline tier to establish governance controls before enabling tuning" and "Switch to the recommended tier configuration."
- **Issue:** The solution scripts do not change tenant state, and Microsoft Learn says eligible tenants have Copilot Tuning enabled by default. The guide should not imply that switching JSON tiers enables or disables Copilot Tuning in the tenant.
- **Recommended correction:** Add a step to inspect Copilot Tuning availability in the Microsoft 365 admin center and explicitly disable or limit tuning there when using the baseline tier. State that JSON tiers document governance posture and do not enforce tenant availability settings.
- **Source URL:** https://learn.microsoft.com/en-us/microsoft-365/copilot/copilot-tuning-admin-guide
- **Source last updated:** 2026-04-13
- **Quote from source:** "If a tenant meets the eligibility threshold, Copilot Tuning appears in the admin center and is enabled by default."

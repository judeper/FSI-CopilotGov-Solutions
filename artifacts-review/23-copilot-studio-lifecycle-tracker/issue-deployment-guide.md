### F1 — Deployment guidance should distinguish scaffold PowerShell from live Power Platform module support

- **Severity:** Major
- **Location:** line 5
- **Current:** The guide says the live integration target assumes Power Platform admin access, an Entra ID app registration, and PowerShell 7.2 or later.
- **Issue:** This conflates the scaffold script runtime with future live Power Platform admin module integration. Microsoft Learn states the Power Apps/Power Platform admin modules use .NET Framework and are incompatible with PowerShell 6.0 and later.
- **Suggested change:** Clarify that repository scripts run in PowerShell 7.2+, but live integration using `Microsoft.PowerApps.Administration.PowerShell` must use Windows PowerShell 5.1 or use REST API/SDK alternatives compatible with PowerShell 7+.
- **Source:** https://learn.microsoft.com/en-us/power-platform/admin/powerapps-powershell (last updated 2026-02-25)
- **Quote:** "PowerShell in this article requires Windows PowerShell version 5.x."

#### Finding F2 — Copilot admin center page capability is overstated and dated

- **Severity:** Major
- **Location:** line 18
- **Current text:** "References the redesigned Microsoft 365 Admin Center Copilot overview page (January 2026), which centralizes security configuration, readiness recommendations, and governance settings as supplemental readiness inputs."
- **Issue:** Current Learn documentation describes Microsoft 365 admin center Copilot settings for scenarios under **Copilot > Settings**, with some scenarios configured there and others acting as shortcuts to other admin centers. It does not support the dated January 2026 redesign claim or the statement that the page centralizes readiness recommendations.
- **Recommended correction:** "References Microsoft 365 admin center Copilot > Settings scenarios, which provide selected Copilot management controls and shortcuts to related admin centers as supplemental readiness inputs."
- **Source URL:** https://learn.microsoft.com/microsoft-365-copilot/microsoft-365-copilot-page
- **Source last updated:** 2026-03-16
- **Quote from source:** "Some scenarios are configured in the admin center, while others are shortcuts to other admin centers."

#### Finding F3 — Copilot licensing model uses non-Learn terminology and hard-coded pricing

- **Severity:** Major
- **Location:** line 102
- **Current text:** "Microsoft 365 Copilot licensing now includes a free Copilot Chat tier (web-grounded, included with E3/E5/Business plans), a paid Microsoft 365 Copilot tier ($30/user/month for Graph-grounded capabilities), and consumption-based Copilot agent credits."
- **Issue:** Microsoft Learn describes Microsoft 365 Copilot as an add-on plan and Microsoft 365 Copilot Chat as included at no extra cost for eligible subscriptions. The hard-coded "$30/user/month" price and "free tier" phrasing are not Learn terminology and may become inaccurate. Pay-as-you-go coverage should reference the documented services instead of a generic "Copilot agent credits" phrase.
- **Recommended correction:** "Account for Microsoft 365 Copilot Chat at no additional cost for eligible subscriptions, Microsoft 365 Copilot as an add-on license, and documented Microsoft 365 Copilot pay-as-you-go services such as Microsoft 365 Copilot Chat, SharePoint agents, and the Microsoft Copilot Retrieval API (Preview)."
- **Source URL:** https://learn.microsoft.com/microsoft-365-copilot/microsoft-365-copilot-licensing; https://learn.microsoft.com/microsoft-365-copilot/pay-as-you-go/overview
- **Source last updated:** 2025-12-01; 2025-06-16
- **Quote from source:** "Web-based chat: Automatically included with an eligible Microsoft 365 subscription at no extra cost."

#### Finding F4 — Copilot Control System admin surfaces are narrowed incorrectly

- **Severity:** Major
- **Location:** line 103
- **Current text:** "The Copilot configuration scanning domain should be updated to reference the Copilot Control System, which provides centralized Copilot feature management, agent governance, and connector oversight in the Microsoft 365 admin center."
- **Issue:** Learn states that Copilot and agent management controls are found mainly across the Microsoft 365 admin center, Power Platform admin center, and Copilot Studio. The current text narrows the Control System to the Microsoft 365 admin center and can misdirect implementers for agent sharing, DLP, and capacity tasks.
- **Recommended correction:** "Reference the Copilot Control System management controls, which span the Microsoft 365 admin center, Power Platform admin center, and Copilot Studio for licensing, agent lifecycle, connector, and customization controls."
- **Source URL:** https://learn.microsoft.com/microsoft-365-copilot/copilot-control-system/management-controls
- **Source last updated:** 2026-02-25
- **Quote from source:** "You can find Copilot and agent management controls mainly in the Microsoft 365 admin center, Power Platform admin center, and Copilot Studio."

### `solutions/01-copilot-readiness-scanner/docs/prerequisites.md`

#### Finding F1 — Copilot administration roles are missing from the role matrix

- **Severity:** Major
- **Location:** lines 21-34
- **Current text:** "The simplest deployment model is to use a Global Administrator account..." followed by workload rows for licensing, Entra identity, Defender, Purview, SharePoint, Teams, and Power Platform.
- **Issue:** The solution includes a Copilot configuration scan domain, but the role matrix does not list the current Copilot admin center roles. Microsoft Learn says AI Administrator can view and make changes to Copilot scenarios in the Microsoft 365 admin center, while Global Reader can view them. Without these roles, least-privilege deployments may fail to access Copilot settings.
- **Recommended correction:** Add a Copilot configuration row such as: "Copilot configuration | AI Administrator for changes or Global Reader / AI Reader for read-only review | Review Microsoft 365 Copilot settings, Control System scenarios, and agent governance visibility." Also keep Global Administrator framed only as a break-glass or controlled exception.
- **Source URL:** https://learn.microsoft.com/microsoft-365-copilot/microsoft-365-copilot-page
- **Source last updated:** 2026-03-16
- **Quote from source:** "To view and make changes to the Copilot scenarios in the Microsoft 365 admin center, sign in with the AI Administrator role."

#### Finding F2 — Licensing Graph scope omits current least-privileged permission

- **Severity:** Minor
- **Location:** lines 42, 53-57
- **Current text:** "Microsoft Graph | `Organization.Read.All`, `Directory.Read.All`, `AuditLog.Read.All`, `Reports.Read.All`" and script scopes that omit `LicenseAssignment.Read.All`.
- **Issue:** The scanner's licensing domain references subscribed SKUs and user license assignments. Microsoft Graph currently lists `LicenseAssignment.Read.All` as the least-privileged delegated/application permission for `GET /subscribedSkus` and delegated permission for `GET /users/{id}/licenseDetails`. The documented scopes are broader and do not call out the least-privileged licensing permission.
- **Recommended correction:** Add `LicenseAssignment.Read.All` to the licensing/Graph permission guidance for live tenant implementations, and clarify when broader `Directory.Read.All` or `Organization.Read.All` is only a higher-privilege fallback.
- **Source URL:** https://learn.microsoft.com/graph/api/subscribedsku-list
- **Source last updated:** 2024-10-30
- **Quote from source:** "Least privileged permissions ... LicenseAssignment.Read.All"

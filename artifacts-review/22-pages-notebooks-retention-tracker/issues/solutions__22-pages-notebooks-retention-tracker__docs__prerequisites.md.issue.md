## Findings

### F1 — Microsoft Copilot prerequisite should use Microsoft 365 Copilot licensing

- **Severity:** Major
- **Location:** lines 5-6
- **Current:** "Microsoft 365 E3 or E5 tenant with Microsoft Copilot..."
- **Issue:** Microsoft Learn names the product/license as Microsoft 365 Copilot and describes it as an add-on plan with prerequisites. The current "Microsoft Copilot" wording is ambiguous and can be confused with consumer Copilot.
- **Suggested change:** State that in-scope users need Microsoft 365 Copilot licensing with a qualifying Microsoft 365 E3/E5 (or equivalent) subscription, plus enabled OneNote, Loop, SharePoint Online, and Copilot Pages/Notebooks where applicable.
- **Source:** https://learn.microsoft.com/en-us/office365/servicedescriptions/office-365-platform-service-description/microsoft-365-copilot (last updated 2026-04-30)
- **Quote:** "Microsoft 365 Copilot is an add-on plan with specific licensing prerequisites."

### F2 — Loop and Copilot Pages read scopes are not documented prerequisites

- **Severity:** Critical
- **Location:** lines 13-16
- **Current:** `Notes.Read.All`; `Sites.Read.All`; "Loop and Copilot Pages read scopes as released by Microsoft for the tenant"
- **Issue:** The OneNote `Notes.Read.All` permission is documented for notebook listing, but Microsoft Learn does not document generic Loop or Copilot Pages read scopes for this inventory. Current Copilot Pages/Notebooks management guidance uses SharePoint Embedded containers, Purview, and Graph export/DriveItem capabilities where documented.
- **Suggested change:** Keep `Notes.Read.All` for OneNote. Replace the Loop/Copilot Pages read-scope bullet with documented prerequisites such as SharePoint Embedded administrator access for container discovery, SharePoint/Graph file permissions where applicable, and Purview permissions for audit/retention lookup.
- **Source:** https://learn.microsoft.com/en-us/microsoft-365/loop/cpcn-loop-purview-management?view=o365-worldwide (last updated 2026-03-16)
- **Quote:** "Sign in to the SharePoint admin center with the SharePoint Embedded administrator role."

### F3 — Purview Compliance Admin is not the documented role name

- **Severity:** Major
- **Location:** lines 41-46
- **Current:** "Purview Compliance Admin for retention-policy lookup"
- **Issue:** Microsoft Learn recommends the Compliance Administrator admin role group for retention policies/labels, or a custom role group with Retention Management/View-Only Retention Management. It also uses Records Management for records-management tasks. "Purview Compliance Admin" is not the documented role name.
- **Suggested change:** Use "Compliance Administrator" or a custom role group with "Retention Management"/"View-Only Retention Management" for retention-policy lookup; use "Records Management" for records/disposition tasks.
- **Source:** https://learn.microsoft.com/en-us/purview/get-started-with-data-lifecycle-management (last updated 2026-04-30)
- **Quote:** "we recommend that you add users to the Compliance Administrator admin role group."

### F4 — Copilot Pages storage location is SharePoint Embedded, not SharePoint or OneDrive containers

- **Severity:** Major
- **Location:** line 58
- **Current:** "retention policies pre-published for the SharePoint and OneDrive containers that host Copilot Pages and OneNote Notebooks"
- **Issue:** Microsoft Learn states Copilot Pages and Copilot Notebooks are stored in a user-owned SharePoint Embedded container. OneNote content can be in SharePoint/OneDrive, but Copilot Pages should be documented separately from OneNote locations.
- **Suggested change:** Reference SharePoint Embedded containers for Copilot Pages, and SharePoint/OneDrive folders or sections for OneNote content. For Purview retention, mention All SharePoint Sites or specific SharePoint Embedded container URLs where applicable.
- **Source:** https://learn.microsoft.com/en-us/microsoft-365/loop/cpcn-storage?view=o365-worldwide (last updated 2026-04-23)
- **Quote:** "Copilot Pages and Copilot Notebooks are stored within your organization in SharePoint Embedded."

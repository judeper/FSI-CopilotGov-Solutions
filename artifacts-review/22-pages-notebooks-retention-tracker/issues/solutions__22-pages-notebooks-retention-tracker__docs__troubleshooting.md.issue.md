## Findings

### F1 — Copilot Pages enumeration troubleshooting assumes an undocumented Graph surface

- **Severity:** Critical
- **Location:** lines 16-22
- **Current:** "Confirm Copilot Pages enumeration is enabled in the tenant..."; "until live Graph access is wired"
- **Issue:** Microsoft Learn does not document a Microsoft Graph Copilot Pages enumeration endpoint. Current guidance for Copilot Pages/Notebooks centers on SharePoint Embedded containers, Purview audit/export, and Graph export/DriveItem capabilities where documented.
- **Suggested change:** Replace this with checks for SharePoint Embedded container discovery, Purview audit/export visibility, and documented Graph DriveItem/export access. Avoid instructing users to enable a non-documented Pages enumeration feature.
- **Source:** https://learn.microsoft.com/en-us/microsoft-365/loop/cpcn-compliance-summary?view=o365-worldwide (last updated 2026-04-28)
- **Quote:** "Graph API export: Supported for third-party tools."

### F2 — OneNote retention troubleshooting should focus on sections and folders

- **Severity:** Major
- **Location:** lines 31-40
- **Current:** "Notebook shows `none` for retention-policy source"; "parent SharePoint site or OneDrive library for an applied retention label"
- **Issue:** Microsoft Learn says retention settings impact OneNote sections as files and pages inherit from those sections. The troubleshooting guidance should not imply direct notebook-level retention assignment.
- **Suggested change:** Troubleshoot OneNote section/folder retention, parent SharePoint/OneDrive policy scope, and retention labels applied to OneNote folders or sections.
- **Source:** https://learn.microsoft.com/en-us/purview/retention-policies-sharepoint (last updated 2026-04-30)
- **Quote:** "Pages from each section are contained within the file and inherit the retention settings from their parent section."

### F3 — Loop API reachability guidance references a non-documented API

- **Severity:** Critical
- **Location:** lines 44-47, 77
- **Current:** "Verify the Loop API is reachable from the deployment host"; "Microsoft Graph, Loop, or Purview endpoints"
- **Issue:** Microsoft Learn describes Loop content storage and administration through SharePoint Embedded, SharePoint/OneDrive, Cloud Policy, SharePoint PowerShell, Purview, and documented Graph export/file surfaces. A standalone Loop API is not documented for this scenario.
- **Suggested change:** Replace Loop API reachability checks with checks for SharePoint Embedded container access, Graph DriveItem/export access where used, Purview audit/retention access, and Loop/Cloud Policy configuration.
- **Source:** https://learn.microsoft.com/en-us/microsoft-365/loop/loop-storage?view=o365-worldwide (last updated 2026-03-16)
- **Quote:** "Loop content is stored in SharePoint, OneDrive, and SharePoint Embedded."

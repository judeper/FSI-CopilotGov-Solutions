## Findings

### F1 — Notebook-retention-log schema should track OneNote sections/folders

- **Severity:** Major
- **Location:** lines 20-32, 77, 79
- **Current:** "OneNote Notebook retention-policy assignment and inheritance lineage" with `notebookId`, `retentionLabel`, and `retentionPolicySource`
- **Issue:** Microsoft Learn states OneNote retention impacts sections as files and pages inherit from sections. The current schema implies direct notebook-level assignment and inheritance.
- **Suggested change:** Revise the schema to include section-level identifiers and parent notebook/container metadata, or explicitly state that notebook-level rows summarize section/folder retention coverage.
- **Source:** https://learn.microsoft.com/en-us/purview/retention-policies-sharepoint (last updated 2026-04-30)
- **Quote:** "Only sections are impacted by the retention settings that you specify."

### F2 — Branching-event-log schema uses undocumented Page lifecycle events

- **Severity:** Major
- **Location:** lines 7, 17-18, 48-60, 80-81
- **Current:** `mutabilityState`; `branchingParentPageId`; `eventType`: `branch`, `fork`, `merge`, `lock`, or `preserve`
- **Issue:** Microsoft Learn documents Copilot Pages/Notebooks version history and audit logs, not branch/fork/merge event types or mutable/locked/preserved Page states.
- **Suggested change:** Replace these fields with documented version-history and Purview audit fields, or label them as sample-only internal lineage states that are not emitted by Microsoft 365.
- **Source:** https://learn.microsoft.com/en-us/microsoft-365/loop/cpcn-compliance-summary?view=o365-worldwide (last updated 2026-04-28)
- **Quote:** "Version History: Export in Purview or via Graph API... 50 versions per file (not configurable)."

### F3 — Copilot Pages retention-label evidence needs limited-support caveat

- **Severity:** Major
- **Location:** lines 15, 73, 77
- **Current:** "retentionLabel: Microsoft Purview retention label assigned to the Page"; "Records retention-label coverage... for Copilot Pages"
- **Issue:** Microsoft Learn says retention labels are supported for Copilot Pages with limited manual application and cannot be viewed or applied directly from a Copilot Page. Record/regulatory record labels also cannot be manually applied from Copilot Pages or Loop.
- **Suggested change:** Add these caveats and distinguish policy coverage from label assignment evidence.
- **Source:** https://learn.microsoft.com/en-us/microsoft-365/loop/cpcn-compliance-summary?view=o365-worldwide (last updated 2026-04-28)
- **Quote:** "Retention labels cannot be viewed or applied directly from a Copilot Page."

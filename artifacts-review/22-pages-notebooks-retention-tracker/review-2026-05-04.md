### `solutions/22-pages-notebooks-retention-tracker/README.md`

#### Finding F1 — Copilot Pages branching and mutability states are not documented lifecycle events

- **Severity:** Major
- **Location:** lines 9, 13, 22, 25, 61-65, 73, 76, 94
- **Current text:** "record branching and mutability lineage events"; "Branching event log"; "Page branching, fork, and mutability transition events"
- **Issue:** Microsoft Learn documents Copilot Pages/Notebooks governance around SharePoint Embedded storage, audit logs, retention, and version history. It does not document branch/fork/merge or mutable/locked/preserved Page states as Microsoft product lifecycle events, so the current wording can lead implementers to build evidence around unsupported signals.
- **Recommended correction:** Replace branch/fork/merge/mutability wording with documented signals such as SharePoint Embedded .page files, Purview audit events, retention labels/policies, and version history. If branch-like lineage remains, label it as an internal sample taxonomy rather than a Microsoft Pages state.
- **Source URL:** https://learn.microsoft.com/en-us/microsoft-365/loop/cpcn-compliance-summary?view=o365-worldwide
- **Source last updated:** 2026-04-28
- **Quote from source:** "Version History: Export in Purview or via Graph API... 50 versions per file (not configurable)."

#### Finding F2 — OneNote retention is section-based, not notebook-level label assignment

- **Severity:** Major
- **Location:** lines 14, 23, 74, 94
- **Current text:** "OneNote Notebook retention-policy coverage and inheritance"; "OneNote Notebook retention-label assignment"
- **Issue:** The README implies that a OneNote notebook itself receives a retention label or policy assignment. Microsoft Learn says retention settings apply to OneNote sections as files; pages inherit from their parent section, and the notebook modified date is not used by Microsoft 365 retention.
- **Recommended correction:** Describe OneNote evidence as section/folder-level retention coverage within the parent SharePoint or OneDrive location, with notebook identity used only as grouping metadata.
- **Source URL:** https://learn.microsoft.com/en-us/purview/retention-policies-sharepoint
- **Source last updated:** 2026-04-30
- **Quote from source:** "Only sections are impacted by the retention settings that you specify."

#### Finding F3 — Loop API wording does not match current Microsoft Learn surfaces

- **Severity:** Major
- **Location:** lines 9, 35, 98
- **Current text:** "wiring live Microsoft Graph, Loop, and Purview data sources"; "Microsoft Graph and Loop API insertion points"
- **Issue:** Current Microsoft Learn pages describe Loop and Copilot Pages/Notebooks content through SharePoint Embedded, SharePoint/OneDrive, Purview, and Graph export/drive surfaces. They do not document a standalone Microsoft Loop API for this inventory pattern.
- **Recommended correction:** Replace "Loop API" with "SharePoint Embedded containers, Microsoft Graph DriveItem/export where documented, Purview audit/retention, and Loop/Cloud Policy admin surfaces."
- **Source URL:** https://learn.microsoft.com/en-us/microsoft-365/loop/loop-storage?view=o365-worldwide
- **Source last updated:** 2026-03-16

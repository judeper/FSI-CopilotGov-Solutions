## Findings

### F1 — Architecture treats Entra featureRolloutPolicies as Copilot rollout state

- **Severity:** Critical
- **Location:** lines 13, 25, 73, 97-101
- **Current:** "The feature collector and platform integrations list `Graph beta rollout policy` / `/policies/featureRolloutPolicies` as sources for Copilot feature state and planned ring updates."
- **Issue:** Microsoft Learn defines featureRolloutPolicy as an Entra ID staged authentication rollout resource. It does not provide Copilot feature inventory, Copilot rings, or Teams/Power Platform Copilot controls.
- **Suggested change:** "Remove Graph featureRolloutPolicies from the Copilot architecture, or isolate it as a non-Copilot Entra staged-authentication example. Document supported Copilot control surfaces separately."
- **Source:** https://learn.microsoft.com/en-us/graph/api/resources/featurerolloutpolicy?view=graph-rest-beta (last updated 2024-07-06)
- **Quote:** "pilot features of Microsoft Entra ID with a specific group before enabling features for entire organization"

### F2 — Teams chat Copilot exposure is shown as Teams admin center policy controlled

- **Severity:** Major
- **Location:** line 27
- **Current:** "`Teams meeting, chat, or app policy settings that control Copilot exposure`"
- **Issue:** Microsoft Teams Learn documentation lists an admin article for Copilot in Teams meetings and events, but no admin article for Copilot in Teams chats and channels. Treating chat Copilot exposure as a Teams admin center policy/export can mislead implementers.
- **Suggested change:** "Limit Teams admin center policy guidance to documented Teams meeting/event and calling controls. Mark Teams chat/channel Copilot inventory as documentation-only unless a current Microsoft Learn admin control is cited."
- **Source:** https://learn.microsoft.com/en-us/microsoftteams/copilot-ai-agents-overview (last updated 2025-04-02)
- **Quote:** "Copilot in Teams chats and channels

None"

### F3 — Power Platform Admin API / environment-level Power Automate Copilot control is overstated

- **Severity:** Major
- **Location:** lines 28 and 75
- **Current:** "`Power Platform Admin API` is described as a source for environment and maker-facing Copilot settings for Power Apps and Power Automate."
- **Issue:** Power Automate Learn guidance says Copilot disablement is tenant-level and environment-level support is not available. The file should not imply a Power Platform Admin API can confirm or enforce environment-level Power Automate Copilot settings.
- **Suggested change:** "Describe Power Platform coverage as Power Platform admin center Copilot settings and documented exports, and call out that Power Automate Copilot disablement is tenant-level where Learn says environment-level support is unavailable."
- **Source:** https://learn.microsoft.com/en-us/power-automate/copilot-overview (last updated 2026-01-16)
- **Quote:** "turn it off only at tenant level. Environment level support isn't available"

### F4 — Domain exclusion and authoritative source controls are not supported by the cited Copilot web-search docs

- **Severity:** Major
- **Location:** lines 112-149
- **Current:** "The web grounding section documents domain exclusion lists, authoritative SharePoint source designation, and drift detection over those settings."
- **Issue:** Microsoft Learn documents `Allow web search in Copilot` in Cloud Policy service as the primary admin control. It does not substantiate the domain blocklist or authoritative source designation controls as Microsoft 365 admin center capabilities.
- **Suggested change:** "Replace the section with documented web-search policy controls, including Cloud Policy service scope and user Web content toggle behavior. Move domain exclusion / authoritative-source text to a future-state or customer-defined pattern with a source requirement."
- **Source:** https://learn.microsoft.com/en-us/microsoft-365/copilot/manage-public-web-access (last updated 2026-02-25)
- **Quote:** "The primary way that IT admins can control access to web search is by using the Allow web search in Copilot policy"

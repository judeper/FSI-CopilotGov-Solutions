## Findings

### F1 — Prerequisites point Copilot implementers to Entra staged rollout API

- **Severity:** Critical
- **Location:** line 28
- **Current:** "`Microsoft Graph beta endpoint /policies/featureRolloutPolicies` is listed as a supported administrative surface."
- **Issue:** The endpoint is documented for Entra ID authentication feature staged rollout, not Microsoft 365 Copilot feature controls. Treating it as a Copilot administrative surface would lead implementers to collect the wrong data.
- **Suggested change:** "Remove `/policies/featureRolloutPolicies` from Copilot prerequisites or explicitly mark it out of scope for Copilot feature management."
- **Source:** https://learn.microsoft.com/en-us/graph/api/resources/featurerolloutpolicy?view=graph-rest-beta (last updated 2024-07-06)
- **Quote:** "pilot features of Microsoft Entra ID with a specific group before enabling features for entire organization"

### F2 — Graph permission list is stale and overbroad for current featureRolloutPolicies

- **Severity:** Major
- **Location:** lines 7 and 16-20
- **Current:** "`Policy.ReadWrite.FeatureRollout`, `Policy.Read.All`, `Directory.Read.All`, and `Reports.Read.All` are listed as minimum Graph scopes."
- **Issue:** The current Microsoft Graph list operation documents `Policy.Read.HybridAuthentication` as the least privileged delegated permission; application permissions are not supported. The listed scopes do not match the current Learn table.
- **Suggested change:** "Do not list these scopes for Copilot FMC. If retaining an Entra staged-rollout appendix, align scopes to the current Microsoft Graph permissions table."
- **Source:** https://learn.microsoft.com/en-us/graph/api/featurerolloutpolicies-list?view=graph-rest-beta (last updated 2024-03-06)
- **Quote:** "Delegated (work or school account) Policy.Read.HybridAuthentication"

### F3 — Power Platform Admin API coverage for Power Automate Copilot is overstated

- **Severity:** Major
- **Location:** line 30
- **Current:** "`Power Platform Admin API or administrative exports for Copilot in Power Apps and Power Automate`"
- **Issue:** Power Automate Learn documentation says Copilot disablement is tenant-level and environment-level support is not available. The prerequisite should not imply API-level environment coverage for Power Automate Copilot settings.
- **Suggested change:** "Reference Power Platform admin center Copilot settings and documented administrative exports; add a caveat for Power Automate tenant-level support."
- **Source:** https://learn.microsoft.com/en-us/power-automate/copilot-overview (last updated 2026-01-16)
- **Quote:** "turn it off only at tenant level. Environment level support isn't available"

## Findings

### F1 — Default graph endpoint is not a Copilot feature-management endpoint

- **Severity:** Critical
- **Location:** lines 76-83
- **Current:** "`graph.endpoint` is `https://graph.microsoft.com/beta/policies/featureRolloutPolicies` with Copilot FMC scopes."
- **Issue:** Microsoft Learn documents this resource as a Microsoft Entra ID staged authentication rollout policy. It cannot serve as a default Copilot feature inventory or ring-management endpoint.
- **Suggested change:** "Remove the endpoint from FMC default configuration or rename it as an out-of-scope Entra staged-rollout reference."
- **Source:** https://learn.microsoft.com/en-us/graph/api/resources/featurerolloutpolicy?view=graph-rest-beta (last updated 2024-07-06)
- **Quote:** "pilot features of Microsoft Entra ID with a specific group before enabling features for entire organization"

### F2 — Default Graph scopes do not match current featureRolloutPolicies permissions

- **Severity:** Major
- **Location:** lines 78-83
- **Current:** "`Policy.Read.All`, `Policy.ReadWrite.FeatureRollout`, `Directory.Read.All`, and `Reports.Read.All`."
- **Issue:** The current Microsoft Graph list operation for featureRolloutPolicies uses `Policy.Read.HybridAuthentication` as least privileged delegated permission and does not support application permissions.
- **Suggested change:** "Remove these scopes from Copilot configuration; if documenting Entra staged rollout separately, update permissions to current Learn values."
- **Source:** https://learn.microsoft.com/en-us/graph/api/featurerolloutpolicies-list?view=graph-rest-beta (last updated 2024-03-06)
- **Quote:** "Delegated (work or school account) Policy.Read.HybridAuthentication"

### F3 — Power Platform Admin API source label is too broad for Power Automate Copilot

- **Severity:** Major
- **Location:** lines 94-100
- **Current:** "`adminApi`: `Power Platform Admin API` for `Power Apps Copilot` and `Power Automate Copilot`."
- **Issue:** Power Automate Learn documentation says Copilot can be turned off only at tenant level in GPU regions and that environment-level support is not available. The config should not imply API-level environment coverage for Power Automate Copilot.
- **Suggested change:** "Change the source label to documented Power Platform admin center Copilot settings/exports and add a tenant-level caveat for Power Automate."
- **Source:** https://learn.microsoft.com/en-us/power-automate/copilot-overview (last updated 2026-01-16)
- **Quote:** "turn it off only at tenant level. Environment level support isn't available"

### F4 — Web grounding governance settings are not documented Copilot admin controls

- **Severity:** Major
- **Location:** lines 135-140
- **Current:** "`domainExclusionEnabled`, `excludedDomains`, `authoritativeSourceSites`, and `webSearchPolicy`."
- **Issue:** Microsoft Learn documents the `Allow web search in Copilot` policy in Cloud Policy service as the primary admin control. It does not substantiate domain exclusion or authoritative source site arrays as Microsoft 365 Copilot admin settings.
- **Suggested change:** "Limit configuration to documented web-search policy state, or mark domain/authoritative-source properties as customer-defined placeholders rather than Microsoft control settings."
- **Source:** https://learn.microsoft.com/en-us/microsoft-365/copilot/manage-public-web-access (last updated 2026-02-25)
- **Quote:** "The primary way that IT admins can control access to web search is by using the Allow web search in Copilot policy"

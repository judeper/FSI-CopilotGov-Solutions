## Findings

### F1 — Label-enumeration troubleshooting lists only delegated permission

- **Severity:** Major
- **Location:** lines 3-13
- **Current:** "Confirm `InformationProtectionPolicy.Read`, `Sites.Read.All`, `Files.Read.All`, and `Mail.Read` are approved where needed."
- **Issue:** The troubleshooting text mentions app registrations but lists only delegated `InformationProtectionPolicy.Read`. Microsoft Graph requires `InformationProtectionPolicy.Read.All` for application access to organization labels.
- **Suggested change:** Add a delegated-vs-application permission table and explicitly include `InformationProtectionPolicy.Read.All` for app-only label enumeration.
- **Source:** https://learn.microsoft.com/graph/api/security-informationprotection-list-sensitivitylabels?view=graph-rest-beta (last updated 2024-04-04)
- **Quote:** "Application InformationProtectionPolicy.Read.All"

### F2 — `assignSensitivityLabel` 403 guidance omits protected API requirements

- **Severity:** Major
- **Location:** lines 15-25
- **Current:** "Confirm Microsoft Purview Information Protection licensing is assigned... Confirm the calling identity is approved..."
- **Issue:** `driveItem: assignSensitivityLabel` is a protected Graph API and requires validation beyond permission consent. The documented least-privileged permissions are `Files.ReadWrite.All` or `Sites.ReadWrite.All`, and support is not documented for US Government or China clouds.
- **Suggested change:** Add protected API validation, least-privileged permissions, long-running operation handling, supported cloud caveats, and file-type/encryption limitations to the 403 troubleshooting steps.
- **Source:** https://learn.microsoft.com/graph/api/driveitem-assignsensitivitylabel?view=graph-rest-1.0 (last updated 2026-04-09)
- **Quote:** "Protected APIs require you to have more validations, beyond permission and consent, before you can use them."

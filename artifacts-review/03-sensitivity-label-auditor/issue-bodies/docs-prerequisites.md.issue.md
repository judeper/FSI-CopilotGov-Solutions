## Findings

### F1 — Licensing names don't match current sensitivity-labeling service description

- **Severity:** Major
- **Location:** lines 5-6
- **Current:** "Microsoft Purview Information Protection P2" and "Microsoft Purview Suite (formerly Microsoft 365 E5 Compliance)"
- **Issue:** Current Microsoft Learn licensing language for sensitivity labeling uses Microsoft 365 E5 and Microsoft 365 Information Protection and Governance / Microsoft Purview Suite combinations. `Microsoft Purview Information Protection P2` is not the documented SKU name.
- **Suggested change:** Reword licensing prerequisites to reference Microsoft 365 E5/A5/G5, Microsoft 365 Information Protection and Governance, Microsoft Purview Suite, or AIP Plan 2 where applicable, and avoid presenting `Microsoft Purview Information Protection P2` as a SKU.
- **Source:** https://learn.microsoft.com/office365/servicedescriptions/microsoft-365-service-descriptions/microsoft-365-tenantlevel-services-licensing-guidance/microsoft-purview-service-description (last updated 2026-04-16)
- **Quote:** "Sensitivity labeling, including automatic or policy‑based labeling, requires a Microsoft 365 E5 license or Microsoft 365 Information Protection and Governance (IPG)."

### F2 — Required roles omit Purview Information Protection role groups

- **Severity:** Major
- **Location:** lines 10-18
- **Current:** "Compliance Administrator", "Security Administrator", "Global Administrator"
- **Issue:** Microsoft Learn lists Information Protection role groups and Sensitivity Label Administrator/Reader options for sensitivity labels. The current list is overbroad and omits least-privilege Purview roles; Global Administrator is not the recommended operating role.
- **Suggested change:** Add Information Protection, Information Protection Admins, Information Protection Analysts/Investigators/Readers, Sensitivity Label Administrator, and Sensitivity Label Reader as appropriate; reserve Global Administrator for break-glass role assignment.
- **Source:** https://learn.microsoft.com/purview/get-started-with-sensitivity-labels (last updated 2026-04-16)
- **Quote:** "For a read-only role, use Sensitivity Label Reader."

### F3 — Application permission for label enumeration is missing `.All`

- **Severity:** Major
- **Location:** lines 20-29
- **Current:** `InformationProtectionPolicy.Read`, `Sites.Read.All`, `Files.Read.All`, `Mail.Read`
- **Issue:** For application access to organization sensitivity labels, Microsoft Graph documents `InformationProtectionPolicy.Read.All`, not `InformationProtectionPolicy.Read`. The current prerequisite can cause app-only label enumeration to fail.
- **Suggested change:** Distinguish delegated and application permissions: delegated `InformationProtectionPolicy.Read`; application `InformationProtectionPolicy.Read.All`; drive label extraction least-privileged `Files.Read.All`; assign-label scenarios `Files.ReadWrite.All` or `Sites.ReadWrite.All`.
- **Source:** https://learn.microsoft.com/graph/api/security-informationprotection-list-sensitivitylabels?view=graph-rest-beta (last updated 2024-04-04)
- **Quote:** "Application InformationProtectionPolicy.Read.All"

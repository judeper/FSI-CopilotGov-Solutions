## Findings

### F1 — `assignSensitivityLabel` needs protected API caveat

- **Severity:** Major
- **Location:** line 136
- **Current:** "The `assignSensitivityLabel` API requires the correct Microsoft 365 and Purview licensing, plus approved permissions for bulk application scenarios."
- **Issue:** The Microsoft Graph API is a protected SharePoint/OneDrive API and requires validations beyond permission consent. The README also omits the least-privileged Graph permissions and cloud availability caveat.
- **Suggested change:** State that `driveItem: assignSensitivityLabel` is a protected API for SharePoint/OneDrive files, requires protected API validation plus `Files.ReadWrite.All` or `Sites.ReadWrite.All`, and is only documented as available in the global service.
- **Source:** https://learn.microsoft.com/graph/api/driveitem-assignsensitivitylabel?view=graph-rest-1.0 (last updated 2026-04-09)
- **Quote:** "Protected APIs require you to have more validations, beyond permission and consent, before you can use them."

### F2 — Label group migration schedule is unsupported by current Learn docs

- **Severity:** Major
- **Location:** line 139
- **Current:** "Forced auto-migration runs December 2025 through May 2026 and is irreversible."
- **Issue:** Current Microsoft Learn guidance describes automatic migration only for specific cases and says migration is gradually rolling out. It does not support the fixed December 2025-May 2026 forced-migration window stated here.
- **Suggested change:** Replace the fixed schedule with: "Label groups are replacing parent labels. Migration is irreversible; automatic migration applies in documented cases, and other tenants should migrate when the Microsoft Purview portal banner is available."
- **Source:** https://learn.microsoft.com/purview/migrate-sensitivity-label-scheme (last updated 2026-04-17)
- **Quote:** "Migration is gradually rolling out, so migrating your parent labels to label groups might not yet be available for your tenant."

### F3 — Sensitivity label definitions are beta-only in Microsoft Graph

- **Severity:** Major
- **Location:** lines 38-39
- **Current:** "Microsoft Graph permissions for label and workload data collection are approved."
- **Issue:** The solution elsewhere pairs label definition collection with Graph `v1.0`, but the `security/informationProtection/sensitivityLabels` list API is documented under Microsoft Graph beta. The README should disclose the beta dependency for label definition enumeration.
- **Suggested change:** Add a prerequisite note that organization sensitivity label enumeration through `/security/informationProtection/sensitivityLabels` currently uses Microsoft Graph beta and should not be described as a production `v1.0` dependency.
- **Source:** https://learn.microsoft.com/graph/api/security-informationprotection-list-sensitivitylabels?view=graph-rest-beta (last updated 2024-04-04)
- **Quote:** "APIs under the `/beta` version in Microsoft Graph are subject to change."

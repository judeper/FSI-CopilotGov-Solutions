## Findings

### F1 — Sensitivity-label definition endpoint is beta, not v1.0

- **Severity:** Critical
- **Location:** lines 341, 353, and 365
- **Current:** `'/security/informationProtection/sensitivityLabels'` included in source endpoints while configuration sets `graphApiVersion` to `v1.0`.
- **Issue:** Microsoft Learn documents list sensitivity labels under Microsoft Graph beta. Implementers using this endpoint with the configured v1.0 version can build a failing collector.
- **Suggested change:** Mark label definition enumeration as a beta dependency, or remove the endpoint from v1.0 source endpoint lists until a v1.0 API is documented.
- **Source:** https://learn.microsoft.com/graph/api/security-informationprotection-list-sensitivitylabels?view=graph-rest-beta (last updated 2024-04-04)
- **Quote:** "APIs under the `/beta` version in Microsoft Graph are subject to change."

### F2 — SharePoint/OneDrive collection should call `extractSensitivityLabels`

- **Severity:** Major
- **Location:** lines 341 and 353
- **Current:** Source endpoints list `'/drives/{drive-id}/items/{item-id}'` and notes say coverage uses drive item metadata and sensitivity label references.
- **Issue:** The documented Graph API for extracting assigned sensitivity labels from files is the `extractSensitivityLabels` action, returning `extractSensitivityLabelsResult` / `sensitivityLabelAssignment`. The current source endpoint list omits the action and may mislead implementers.
- **Suggested change:** Use `POST /drives/{drive-id}/items/{item-id}/extractSensitivityLabels` (and equivalent site/user path forms) for supported SharePoint/OneDrive files, with `Files.Read.All` for read-only extraction.
- **Source:** https://learn.microsoft.com/graph/api/driveitem-extractsensitivitylabels?view=graph-rest-1.0 (last updated 2024-04-04)
- **Quote:** "POST /drives/{drive-id}/items/{item-id}/extractSensitivityLabels"

### F3 — Exchange message source endpoint has no documented sensitivity-label field

- **Severity:** Major
- **Location:** line 365
- **Current:** `'/users/{user-id}/messages'` with coverage notes stating Exchange coverage is derived from message-level label metadata.
- **Issue:** The Microsoft Graph `message` resource does not document a first-class sensitivity label or `contentLabel` field. This script should not imply `/users/{user-id}/messages` alone provides Purview sensitivity label coverage.
- **Suggested change:** Replace or qualify this endpoint with the tenant-approved Exchange/Purview source used for message label evidence, such as audit/activity data or documented extended-property extraction.
- **Source:** https://learn.microsoft.com/graph/api/resources/message?view=graph-rest-1.0 (last updated 2024-08-23)
- **Quote:** "The following JSON representation shows the resource type."

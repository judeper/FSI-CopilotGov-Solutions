## Findings by file

### `solutions/15-pages-notebooks-gap-monitor/README.md`

#### Finding F1 — Copilot Pages compliance framing is outdated and overbroad

- **Severity:** Major
- **Location:** lines 15-16
- **Current text:** "Copilot Pages are fluid, Loop-based documents... Microsoft Loop workspaces and Copilot Pages content may not receive the same retention, Microsoft Purview eDiscovery, or books-and-records coverage..."
- **Issue:** Current Microsoft Learn guidance says Copilot Pages and Copilot Notebooks are independent of Loop and stored in SharePoint Embedded, and that retention policies and Purview eDiscovery are supported with specific limitations. The current text overstates unsupported coverage and uses the older "Loop-based" framing.
- **Recommended correction:** Describe Copilot Pages as `.page` files stored in user-owned SharePoint Embedded containers. State that Purview retention policies and eDiscovery are supported for Copilot Pages/Notebooks, then list the current limitations: no full-text search in review sets, manual legal-hold container addition, limited manual retention-label support, and no Information Barriers for SharePoint Embedded content.
- **Source URL:** https://learn.microsoft.com/en-us/microsoft-365/loop/cpcn-compliance-summary
- **Source last updated:** 2026-04-28
- **Quote from source:** "Retention policies from Microsoft Purview Data Lifecycle Management configured for all SharePoint sites are enforced for all Copilot Pages and Copilot Notebooks."

### `solutions/15-pages-notebooks-gap-monitor/docs/architecture.md`

#### Finding F2 — Architecture still says native retention/eDiscovery/audit coverage is not available

- **Severity:** Major
- **Location:** lines 5, 95-102
- **Current text:** "...notebook experiences do not yet provide the books-and-records, retention, Microsoft Purview eDiscovery, or audit coverage..." and "Loop workspaces can have limited Microsoft Purview eDiscovery search scope..."
- **Issue:** Microsoft Learn now documents supported retention policies, supported Purview eDiscovery, supported legal hold with manual container addition, and audit logs for Copilot Pages/Notebooks and Loop. The architecture should focus on the remaining limitations rather than broad missing native coverage.
- **Recommended correction:** Replace the broad unsupported-coverage claim with a current limitation list: retention policies via All SharePoint Sites are supported, eDiscovery search/collection/review/export is supported, full-text search in review sets is unavailable, legal hold requires adding SharePoint Embedded containers per user, retention-label support is limited, and Information Barriers are not supported for SharePoint Embedded content.
- **Source URL:** https://learn.microsoft.com/en-us/microsoft-365/loop/cpcn-compliance-summary
- **Source last updated:** 2026-04-28
- **Quote from source:** "eDiscovery ✅ Supported (full-text search in review sets not available)"

### `solutions/15-pages-notebooks-gap-monitor/docs/prerequisites.md`

#### Finding F3 — Copilot Pages license requirement is overstated

- **Severity:** Major
- **Location:** line 5
- **Current text:** "Microsoft 365 Copilot licenses for users whose Pages or Notebooks content is in scope"
- **Issue:** Microsoft Learn distinguishes licensing requirements: Copilot Pages requires a OneDrive license with an active OneDrive site, while Copilot Notebooks requires a Microsoft 365 Copilot license. The current prerequisite can cause teams to over-license Pages-only users.
- **Recommended correction:** Replace with: "OneDrive license and active OneDrive site for Copilot Pages; Microsoft 365 Copilot license for Copilot Notebooks."
- **Source URL:** https://learn.microsoft.com/en-us/microsoft-365/loop/cpcn-requirements
- **Source last updated:** 2026-04-29
- **Quote from source:** "Users need a OneDrive license and an active OneDrive site."

#### Finding F4 — InformationProtectionPolicy permission is mischaracterized

- **Severity:** Major
- **Location:** lines 21-25
- **Current text:** "`InformationProtectionPolicy.Read` for retention and label policy inspection"
- **Issue:** The cited Microsoft Graph permission applies to beta sensitivity-label APIs. For app-only access the permission is `InformationProtectionPolicy.Read.All`, and the API is not a general retention-policy inventory permission. The current prerequisite would mislead future authenticated implementation work.
- **Recommended correction:** Qualify the permission as beta sensitivity-label lookup only, use `InformationProtectionPolicy.Read.All` for application permissions if client-secret/app-only access is implemented, and document a separate supported Purview/PowerShell approach for retention policy inventory.
- **Source URL:** https://learn.microsoft.com/en-us/graph/api/security-informationprotection-list-sensitivitylabels
- **Source last updated:** 2025-11-13
- **Quote from source:** "InformationProtectionPolicy.Read.All"

### `solutions/15-pages-notebooks-gap-monitor/docs/troubleshooting.md`

#### Finding F5 — Loop eDiscovery troubleshooting describes an outdated unsupported-search condition

- **Severity:** Major
- **Location:** lines 28-38
- **Current text:** "Loop workspace content is not yet surfaced consistently for the search workflow being used."
- **Issue:** Microsoft Learn now states Purview eDiscovery is supported for Loop, with the specific limitation that full-text search within `.loop` files in review sets is unavailable. The troubleshooting guidance should direct operators to the documented limitation and container scoping, not a broad unsupported-surfacing condition.
- **Recommended correction:** Reframe the possible cause as incorrect case scope/container URL or the documented full-text review-set limitation, and state that Purview eDiscovery supports Loop search/collection, review, and export with licensing requirements.
- **Source URL:** https://learn.microsoft.com/en-us/microsoft-365/loop/loop-compliance-summary
- **Source last updated:** 2026-04-28
- **Quote from source:** "Purview eDiscovery: Supported for search/collection, review (Premium license required), and export as HTML"

- **Current text:** "Copilot Pages stored in Loop-backed workspaces may not consistently inherit tenant retention coverage..." and "Loop workspace content referenced by Copilot Pages may not appear consistently in Microsoft Purview eDiscovery workflows..." with `platformUpdateRequired = $true`.

- **Quote from source:** "Purview eDiscovery: Supported for search/collection, review (Premium license required), and export as HTML"
